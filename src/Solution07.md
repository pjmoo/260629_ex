# Solution07: 객체 정렬과 Comparator

## 1. 초심자용 비유로 이해하기

### ❶ 정렬(Sort) vs 정렬(Align)

* **비유**: **"키순으로 줄 세우기(Sort) vs 칠판 좌우 정렬하기(Align)"**
* **상세**:
    * **Sort(정렬)**: 여러 데이터를 하나의 명확한 기준(키, 몸무게, 이름 가나다순 등 1차원 기준)에 따라 순서대로 나열하는 것을
      말합니다. [Solution07.java](file:///Users/morgan/Documents/workspace/260629_ex/src/Solution07.java)에서 배우는 정렬이 바로 이
      `Sort`입니다.
    * **Align(배치 정렬)**: 화면 안에서 텍스트나 이미지 요소를 왼쪽, 가운데, 오른쪽에 깔끔하게 배치하는 UI적인 정렬을 뜻합니다.

### ❷ `Comparator` (비교 판정기)

* **비유**: **"두 명을 비교해 순서를 결정하는 줄 세우기 도우미"**
* **상세**: 리스트 안의 모든 원소를 한꺼번에 정렬하는 것은 복잡하지만, 결국 정렬은 **"두 원소(e1, e2) 중 누가 앞에 서야 하는가?"**를 반복해서 결정하는 과정입니다.
    * `(e1, e2) -> e1.a() - e2.a()` 와 같은 람다식은 줄 세우기 도우미에게 주는 **비교 규칙**입니다.
    * 도우미는 두 명을 데려와 뺄셈 연산을 해봅니다.
        * 뺀 결과가 **양수(+)**이면, 앞뒤 순서를 바꿉니다. (뒤의 값이 더 작다는 뜻이므로 오름차순 정렬 시 앞뒤 체인지)
        * 뺀 결과가 **음수(-)**이거나 **0**이면, 그대로 둡니다.
    * 절대값 정렬(`Math.abs(e1) - Math.abs(e2)`)이나 객체의 특정 필드 비교(`e1.a() - e2.a()`) 모두 이 간단한 뺄셈 규칙을 이용해 복잡한 객체 정렬을 가능하게 만듭니다.

## 2. 예상 면접 질문 & 모범 답변

### Q1. 자바에서 객체를 정렬할 때 사용하는 `Comparable`과 `Comparator` 인터페이스의 차이는 무엇인가요?

* **답변**:
    * **`Comparable`**: 객체의 **기본 정렬 기준**을 정의할 때 사용합니다. 클래스가 `Comparable<T>`을 구현하고 `compareTo` 메서드를 오버라이드하여 내부에 직접
      구현합니다. (예: `String`의 사전순, `Integer`의 오름차순)
    * **`Comparator`**: 기본 정렬 기준 외에 **새로운 정렬 기준**을 동적으로 정의하고 싶을 때 사용합니다. 정렬 메서드(`List.sort`나 `Collections.sort`)의 매개변수로
      `Comparator` 구현체나 람다식을 전달하여 사용합니다.

### Q2. `Comparator`가 반환하는 값(양수, 0, 음수)의 의미는 정렬에 어떤 영향을 주나요?

* **답변**: 두 원소 `e1`과 `e2`를 비교할 때 반환되는 정수값에 따라 순서가 결정됩니다.
    * **양수(Positive)**: 첫 번째 인자(`e1`)가 두 번째 인자(`e2`)보다 정렬 순서상 뒤에 위치해야 함을 의미하며, 오름차순 기준으로 두 원소의 위치를 바꿉니다.
    * **음수(Negative)**: 첫 번째 인자(`e1`)가 두 번째 인자(`e2`)보다 앞에 위치해야 함을 의미하며, 순서를 그대로 유지합니다.
    * **0**: 두 원소의 정렬 순서가 동일함을 의미합니다.

### Q3. `Comparator.comparing`이나 `comparingInt`를 사용해 객체 정렬을 구현하는 것의 장점은 무엇인가요?

* **답변**: 직접 뺄셈 연산(`e1.a() - e2.a()`)을 작성하는 대신, 객체에서 정렬 기준이 될 필드를 추출하는 메서드 참조(예: `Container::a`)만 전달하면 되므로 코드가 훨씬 선언적이고
  가독성이 향상됩니다. 또한 직접 뺄셈을 할 때 발생할 수 있는 정수 오버플로우/언더플로우 문제를 방지해 주어 더 안전합니다.
