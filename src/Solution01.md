# Solution01: 동시성 기초, Race Condition, 스택 메모리 격리

## 1. 초심자용 비유로 이해하기

### ❶ 레이스 컨디션 (Race Condition)
* **비유**: **"하나의 공용 메모장에 두 명이 동시에 낙서하기"**
* **상세**: 칠판에 숫자 `0`이 적혀 있습니다. 철수와 영희에게 각각 "칠판의 숫자를 읽고, 거기에 1을 더해서 다시 적어라"라는 심부름을 시켰습니다.
    1. 철수가 칠판을 보니 `0`입니다. 머릿속으로 `0 + 1 = 1`을 계산합니다.
    2. 철수가 지우개로 지우고 칠판에 `1`을 쓰려는 찰나, 영희가 칠판을 봅니다. 아직 철수가 적기 전이라 영희 눈에도 `0`으로 보입니다. 영희도 머릿속으로 `0 + 1 = 1`을 계산합니다.
    3. 철수가 칠판에 `1`을 씁니다.
    4. 바로 뒤이어 영희도 자기 머릿속 계산 결과인 `1`을 씁니다.
    * **결과**: 두 명이 각각 1번씩 총 2번 더했으므로 최종 숫자는 `2`가 되어야 하지만, 동기화가 되지 않아 최종 숫자는 `1`이 되어 버렸습니다. 이것이 바로 데이터가 유실되는 **레이스 컨디션(경쟁 상태)**입니다.

### ❷ 스택 메모리 격리 (Stack Isolation)
* **비유**: **"각자 지급받은 개인 도마와 칼"**
* **상세**: 공용 주방(Heap 메모리)에서 요리를 할 때, 공용 냄비(공유 자원)를 같이 쓰면 서로 내용물이 섞여 엉망이 됩니다. 하지만 요리사마다 하나씩 지급받은 개인 도마(Stack 메모리) 위에서 채소를 써는 일은 옆 사람의 영향을 전혀 받지 않습니다.
* [Solution01.java](file:///Users/morgan/Documents/workspace/260629_ex/src/Solution01.java#L31)의 `localCounter`처럼 메서드 내부에 선언된 지역 변수는 스레드마다 독립적인 스택 메모리에 할당되므로, 멀티스레드 환경에서도 안전하게 본인만의 값을 유지할 수 있습니다.

## 2. 예상 면접 질문 & 모범 답변

### Q1. 경쟁 상태(Race Condition)란 무엇이며, 자바에서 이를 방지하는 방법은 무엇인가요?
* **답변**: 경쟁 상태는 여러 스레드가 동시에 공유 자원에 접근하여 읽고 쓰는 작업을 할 때, 실행 순서에 따라 결과가 달라지거나 데이터가 유실되는 현상을 말합니다. 자바에서는 `synchronized` 키워드 사용, `Lock` 인터페이스 구현체(`ReentrantLock`) 활용, 혹은 `Atomic` 계열 클래스를 사용하여 원자적(Atomic) 연산을 보장함으로써 경쟁 상태를 방지할 수 있습니다.

### Q2. 힙(Heap) 메모리와 스택(Stack) 메모리의 멀티스레드 관점에서의 차이를 설명해 주세요.
* **답변**:
    * **힙 메모리**: 모든 스레드가 공유하는 영역입니다. 객체(인스턴스)와 인스턴스 변수가 여기에 저장되며, 여러 스레드가 동시에 접근해 수정할 수 있어 동기화 처리가 필요합니다.
    * **스택 메모리**: 각 스레드마다 독립적으로 할당되는 영역입니다. 메서드 호출 시 생성되는 지역 변수와 매개변수가 저장되며, 해당 스레드만 접근할 수 있기 때문에 별도의 동기화 없이도 스레드 안전(Thread-safe)합니다.

### Q3. 지역 변수(Local Variable)는 왜 멀티스레드 환경에서 안전한가요?
* **답변**: 지역 변수는 각 스레드가 메서드를 실행할 때 자신만의 독립적인 스택 프레임(Stack Frame) 내에 생성되기 때문입니다. 다른 스레드와 메모리를 공유하지 않으므로 레이스 컨디션이 발생할 여지가 없어서 항상 스레드 안전합니다.
