# Solution06: 람다식, 함수형 인터페이스, 변수 캡처, 메서드 참조

## 1. 초심자용 비유로 이해하기

### ❶ 익명 클래스 vs 람다식 (Lambda)

* **비유**: **"변호사 선임 계약서 vs 포스트잇 메모"**
* **상세**:
    * **익명 클래스**: 아주 간단한 심부름 하나를 시키기 위해 온갖 서류 양식(클래스 정의, 오버라이드 어노테이션, 메서드 시그니처 등)을 다 작성해서 계약서 서명을 받아 수행하는 공식적인 방식입니다.
      코드가 너무 길어져 가독성이 떨어집니다.
    * **람다식**: 포스트잇에 딱 한 줄로 **`(msg) -> System.out.println(msg)`** 처럼 "할 일(코드 자체)"만 명확하고 간결하게 적어서 건네는 방식입니다. 불필요한
      보일러플레이트 코드가 사라져 핵심 로직만 한눈에 파악할 수 있습니다.

### ❷ 변수 캡처 (Variable Capture)

* **비유**: **"칠판에 적힌 계획표를 사진 찍어 가져가기"**
* **상세**: 람다식은 자신이 만들어진 장소 주변의 지역 변수를 가져와 쓸 수 있습니다. 이를 **변수 캡처**라고 합니다.
* 그런데 람다를 들고 다른 스레드로 멀리 떠나 일을 하려는데, 원본 칠판의 값(지역 변수 값)이 중간에 바뀌어 버리면 람다가 가지고 있는 사진 속 정보와 일치하지 않아 데이터 정합성이 깨집니다.
* 따라서 자바는 람다가 캡처해서 사용하는 지역 변수는 반드시 **상수(`final`)**이거나 값 변경이 없는 **사실상 상수(`effectively final`)**여야만 하도록 법적으로 제한해 둡니다. 단,
  배열(`arr[0]++` 등)이나 힙 영역 객체의 내부 상태 변경은 원본 주소값 자체(배열의 참조 주소)가 변하지 않기 때문에 허용됩니다.

### ❸ 메서드 참조 (Method Reference)

* **비유**: **"백종원 레시피 적어주기 vs 백종원 책의 몇 페이지 보라고 하기"**
* **상세**: 람다식마저도 이미 작성되어 있는 다른 메서드의 행동과 완전히 같다면 굳이 포스트잇에 그 행동을 일일이 적을 필요가 없습니다.
* `(text) -> System.out.println(text)`라고 쓰는 대신, **`System.out::println`**처럼 "그냥 System.out의 println 메서드 쓰면 돼"하고 좌표(참조
  주소)만 콕 찝어 주는 것이 메서드 참조입니다. 코드가 극도로 간결해집니다.

## 2. 예상 면접 질문 & 모범 답변

### Q1. 함수형 인터페이스(Functional Interface)란 무엇이며, `@FunctionalInterface` 어노테이션의 역할은 무엇인가요?

* **답변**: 함수형 인터페이스는 단 **하나의 추상 메서드(Single Abstract Method, SAM)**만 가지는 인터페이스를 말합니다.
    * 자바는 람다식을 통해 함수형 인터페이스의 구현체를 간결하게 인스턴스화할 수 있도록 지원합니다.
    * `@FunctionalInterface` 어노테이션은 컴파일러에게 해당 인터페이스가 함수형 인터페이스의 규칙(추상 메서드가 1개만 존재하는지)을 올바르게 지켰는지 검증하도록 강제합니다. 만약 2개 이상의
      추상 메서드가 선언되거나 하나도 없으면 컴파일 에러를 발생시킵니다.

### Q2. 람다식 내에서 참조하는 지역 변수는 왜 반드시 `final`이거나 `effectively final`이어야 하나요?

* **답변**: 자바의 지역 변수는 스레드마다 독립적인 스택 메모리에 존재하며, 메서드 실행이 끝나면 스택 프레임과 함께 소멸합니다.
    * 하지만 람다식은 스레드 풀을 타고 다른 스레드에서 원본 메서드 생명주기보다 더 오래 실행될 수 있습니다.
    * 이때 원본 변수가 사라지는 문제를 막기 위해 자바는 원본 지역 변수의 복사본을 람다식 인스턴스 내부에 가지고 있게 되는데, 이를 **변수 캡처**라고 합니다.
    * 이때 복사된 값과 원본 지역 변수의 값이 도중에 서로 달라지면 데이터 정합성이 깨지며 멀티스레드 환경에서 심각한 레이스 컨디션을 유발할 수 있으므로, 해당 지역 변수는 오직 읽기만 가능한 `final`
      또는 `effectively final`(사실상 상수) 상태여야 합니다.

### Q3. 메서드 참조(Method Reference)의 4가지 유형은 무엇인가요?

* **답변**: 메서드 참조는 람다식이 단일 메서드 호출로만 이루어져 있을 때 코드를 더욱 간결하게 표현하는 문법입니다. 유형은 다음과 같습니다.
    1. **정적(Static) 메서드 참조**: `ClassName::staticMethodName` (예: `Integer::parseInt`)
    2. **특정 객체의 인스턴스 메서드 참조**: `instanceReference::instanceMethodName` (예: `System.out::println`)
    3. **임의 객체의 인스턴스 메서드 참조**: `ClassName::instanceMethodName` (예: `String::toUpperCase`)
    4. **생성자 참조**: `ClassName::new` (예: `ArrayList::new`)
