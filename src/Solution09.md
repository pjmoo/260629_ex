# Solution09: 스트림 연산(Map, Filter, Reduce) 및 병렬 스트림(Parallel Stream)

## 1. 초심자용 비유로 이해하기

### ❶ 맵 (Map)

* **비유**: **"사과 껍질 벗기기 기계"**
* **상세**: 5개의 사과가 기계로 들어가면 껍질이 벗겨진 5개의 사과가 나옵니다. 개수(부피)는 변하지 않지만, 각 물건의 형태나 종류가 다른 것으로 변환됩니다.
* [Solution09.java](file:///Users/morgan/Documents/workspace/260629_ex/src/Solution09.java#L40-L45)의 `map` 연산은 가수를 나타내는
  `Singer` 객체를 받아 나이에 따라 `"ADULT"` 혹은 `"CHILD"`라는 문자열로 형태를 탈바꿈시켰습니다.

### ❷ 필터 (Filter)

* **비유**: **"크기 분류용 그물망"**
* **상세**: 작은 물고기는 빠져나가고 큰 물고기만 남기는 그물망과 같습니다. 조건에 부합하는 원소들만 걸러내어 통과시키기 때문에 결과물의 개수가 줄어들 수 있습니다.
* 코드에서 나이가 18세 초과인 가수들만 걸러내는 `.filter((el) -> el.age > 18)` 연산이 이에 해당합니다.

### ❸ 리듀스 (Reduce)

* **비유**: **"눈사람 만들기 (눈덩이 굴리기)"**
* **상세**: 흩어져 있는 눈송이(컬렉션의 원소들)를 하나로 뭉쳐 큰 눈덩이 하나를 만드는 작업입니다.
* 코드의 `.reduce(0, (prev, cur) -> prev + cur.age, (el1, el2) -> el1 + el2)`는 다음 3단계 부품으로 이루어져 있습니다.
    1. **초기값(Identity - `0`)**: 맨 처음 뭉칠 조그만 눈뭉치 씨앗입니다.
    2. **누적기(Accumulator - `(prev, cur) -> ...`)**: 기존에 뭉쳐진 눈덩이(`prev`)에 새로운 눈(`cur.age`)을 붙여 더 크게 굴려 나가는 동작입니다.
    3. **결합기(Combiner - `(el1, el2) -> ...`)**: 눈사람을 빨리 만들기 위해 여러 사람(여러 스레드)이 각자 구역에서 굴린 눈덩이들을 최종적으로 하나로 뭉쳐 합치는(
       `el1 + el2`) 동작입니다. (병렬 처리 시 필수적입니다.)

### ❹ 병렬 스트림 (Parallel Stream)과 경합 문제

* **비유**: **"여러 명의 배달부와 하나의 좁은 우편함"**
* **상세**: 한 명의 배달부(싱글 스레드)가 우편함에 편지를 차곡차곡 넣을 때는 아무 문제없습니다. 하지만 배달 속도를 높이기 위해 10명의 배달부(병렬 스레드)를 고용해서 하나의 좁은 우편함(동기화 처리가 안
  된 `ArrayList`)에 편지를 동시에 마구 쑤셔 넣으려고 하면(`list::add`), 배달부들이 부딪혀 편지가 길거리에 유실되거나 우편함이 찌그러지는 참사(자료구조가 깨져서 에러 발생 혹은 누락)가
  발생합니다.
* 따라서 [Solution09.java](file:///Users/morgan/Documents/workspace/260629_ex/src/Solution09.java#L16-L28)
  의 [runParallel](file:///Users/morgan/Documents/workspace/260629_ex/src/Solution09.java#L16)처럼 병렬 처리를 할 때는 공용 리스트에 직접
  원소를 밀어 넣는 방식(`forEach(list::add)`) 대신, 안전하게 수집을 제어하는 최종 연산인 `.toList()` 등을 사용하여 최종 결과를 안전하게 모아야 합니다.

## 2. 예상 면접 질문 & 모범 답변

### Q1. 스트림의 `reduce()` 연산은 무엇이며, 3개의 매개변수를 받는 메서드 시그니처의 각 인자는 어떤 역할을 하나요?

* **답변**: `reduce()`는 스트림의 많은 원소들을 하나의 값으로 결합(축소)하는 폴딩(Folding) 연산입니다.
    * **첫 번째 인자 (Identity)**: 연산의 초기값입니다. 스트림이 비어 있어도 이 값이 반환됩니다.
    * **두 번째 인자 (Accumulator)**: 각 원소를 차례대로 돌며 이전 누적값과 현재 원소를 조합하여 새로운 값을 계산하는 함수형 인터페이스(`BiFunction`)입니다.
    * **세 번째 인자 (Combiner)**: 병렬 스트림(Parallel Stream) 환경에서 여러 스레드가 각자 연산한 부분합 결과들을 최종적으로 엮어 하나의 결과로 병합하는 함수형 인터페이스(
      `BinaryOperator`)입니다.

### Q2. 병렬 스트림(Parallel Stream)을 사용할 때 성능이 오히려 하락하거나 버그가 발생하는 원인은 무엇인가요?

* **답변**:
    * **성능 하락**: 데이터의 개수가 너무 적거나, 각 원소당 처리 비용이 낮을 때, 또는 스트림 소스가 분할하기 어려운 구조(`LinkedList` 등)일 경우 스레드 풀 생성 및 관리 비용, 그리고 작업
      분할 및 병합 오버헤드가 더 커져 오히려 느려질 수 있습니다.
    * **버그 발생 (Thread-safety 문제)**: 여러 스레드가 동시에 공유 가변 자원에 접근하여 상태를 수정할 때 발생합니다. 예컨대 병렬 스트림 내부에서 동기화되지 않은 `ArrayList`에
      `.forEach(list::add)`로 값을 직접 추가하는 경우, 데이터 유실(Data loss)이나 `ArrayIndexOutOfBoundsException`이 발생할 수 있습니다.

### Q3. 병렬 스트림에서 안전하게 컬렉션으로 결과를 수집하려면 어떻게 해야 하나요?

* **답변**: 병렬 스트림 연산 시 공유 객체의 상태를 직접 변경(Side-effect)하는 `forEach(list::add)` 같은 구문은 지양해야 합니다. 대신 리듀싱 연산의 스레드 안전성을 내부적으로
  보장해 주는 `.collect(Collectors.toList())`나 `.toList()`와 같은 최종 연산을 사용하여 리턴된 결과를 수집해야 합니다.
